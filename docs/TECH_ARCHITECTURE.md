# Technical Architecture

## 总体架构

Image Sifter 是一个 Electron 桌面应用。主进程负责访问本地文件系统、打开文件夹选择器、将文件送到系统回收站；渲染进程负责显示界面、接收按钮和键盘操作。

## 模块拆分

- `src/main/main.js`：创建窗口，注册 IPC 接口，处理文件夹选择、图片列表读取、回收站删除。
- `src/main/preload.js`：向渲染进程暴露安全的 `imageSifter` API。
- `src/renderer/index.html`：应用界面结构。
- `src/renderer/styles.css`：应用视觉样式。
- `src/renderer/app.js`：界面状态、图片切换、快捷键处理。
- `src/shared/imageFiles.js`：图片扩展名判断和图片文件排序。
- `src/shared/navigator.js`：上一张、下一张、删除后跳转等纯逻辑。
- `tests/`：核心逻辑测试。

## 技术栈选择

- Electron：提供 Windows 桌面窗口、本地文件夹选择、系统回收站能力。
- 原生 HTML/CSS/JavaScript：MVP 交互简单，不引入前端框架。
- Node.js test runner：测试图片筛选和导航逻辑。

## 数据流

1. 渲染进程请求选择文件夹。
2. 主进程打开系统文件夹选择器。
3. 主进程读取文件夹内图片文件，并返回图片路径、文件 URL 和文件名。
4. 渲染进程保存图片列表和当前索引。
5. 用户切换图片时，渲染进程更新当前索引。
6. 用户删除图片时，渲染进程请求主进程将当前路径送到回收站。
7. 主进程删除成功后返回结果，渲染进程从列表移除该图片并显示下一张。

## 输入输出格式

主进程返回的图片对象：

```json
{
  "path": "D:\\Pictures\\example.jpg",
  "url": "file:///D:/Pictures/example.jpg",
  "name": "example.jpg"
}
```

## 接口设计

通过 preload 暴露：

- `selectFolder()`：打开文件夹选择器并返回图片列表。
- `trashImage(filePath)`：将图片送到系统回收站。

## 文件存储方式

程序不创建数据库，不复制图片，不上传图片。图片保留在原文件夹；删除时进入操作系统回收站。

## 是否需要数据库

不需要。

## 是否需要 API

不需要外部 API。

## 是否需要登录或授权

不需要。

## 哪些部分由代码完成

文件读取、图片筛选、界面显示、快捷键响应、回收站删除全部由本地代码完成。

## 哪些部分由 AI 模型完成

MVP 不调用 AI 模型。

## 错误处理方式

- 用户取消选择文件夹：保持空状态。
- 文件夹没有图片：显示空状态。
- 图片删除失败：显示错误消息，不从列表移除。
- 图片加载失败：显示错误提示并允许切换下一张。

## 后续扩展接口

后续可以在主进程扩展递归扫描、撤销删除、自定义快捷键持久化、缩略图缓存等能力。
